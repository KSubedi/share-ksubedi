ALEX: The Trusted Runtime for Local AI.

The Product

AI That Works Where the Data Lives

ALEX is a local-first AI platform that operates the computer, serves intelligence to every app, and enforces enterprise policy, all from one install.

ALEX can run fully air-gapped, with no internet connection, as well as in local, hybrid, or cloud configurations. It reasons over documents, uses tools, and runs workflows where the data lives. Cloud use is always an explicit choice, never a silent dependency.

This unlocks capable AI for critical environments where conventional cloud agents cannot be used, including governance, safety-critical operations, privacy-sensitive work, healthcare, and other regulated domains.

At its core is MARS, the Model Assisted Runtime Supervisor. MARS guides local models through edge cases, performs safety-critical checks before actions run, and dynamically routes work to the right model and environment. This supervisory layer makes capable local AI practical for real work.

ALEX does not remain a static assistant. It evolves automatically with each user: learning from conversations and workflows, remembering what matters, clearing and updating stale memories, and creating or improving Skills and scripts as needs emerge. The result is a private agent that becomes more capable and personalized through use, without requiring the user to program it.

The Moat

Defensibility Compounds at Three Layers

The moat is not a single model. It is the system around the models.

Proprietary supervision. MARS turns model limitations into managed runtime decisions: guiding edge cases, checking safety, and routing work across local and cloud models. Its value compounds as evaluations and deployments expand.

Compounding agent evolution. ALEX continuously learns from conversations and workflows, curates its own memory, and creates and updates Skills and scripts automatically. Every interaction improves the agent and deepens its fit with the user or organization, creating a proprietary context and workflow layer that becomes increasingly difficult to replace.

Deployment and trust infrastructure. Reliable local agents require deep product work across tool use, recovery, hardware variation, policy, auditability, and disconnected operation. ALEX packages that know-how into a working product for environments where data cannot leave the organization or internet access is prohibited.
