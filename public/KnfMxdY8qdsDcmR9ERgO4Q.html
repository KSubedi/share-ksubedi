Law Nepal Project — Smaller Model Run Report

Setup

You ran the Gemma-3 Nepali legal model (gemma-3-nepali-legal-updated) locally via a Fastify inference server (local-inference-server.py) on http://127.0.0.1:8083/v1/chat/completions. The model is a 4-bit quantized version of unsloth/gemma-3-4b-it (NF4 quantization via bitsandbytes), fine-tuned with LoRA on 21,869 bilingual Nepali legal/dictionary pairs, then merged. The model files sit in /Users/kaushal/workspace/law-nepal/model/ — the safetensors checkpoint is ~4.5 GB, plus a 2.4 GB Q4_K_M GGUF copy.

The .env.local-model config pointed the eval harness at the local server with OPENROUTER_API_KEY=local and EVAL_USE_DIRECT=true (non-agentic mode).

What was tested

The eval ran on one case: faujadaree/case_02 — a कुटपिट (assault) case from the Purna Path dataset. The model was asked to generate a ठहर खण्ड (legal reasoning) and तपसिल खण्ड (directives/orders) section, matching the format of real Nepali district court judgments.

What went wrong

  1. Generator output was incoherent — the model produced repetitive, nonsensical Nepali text instead of structured legal reasoning. The best run (14% local similarity) was a 1985-token ramble about "internal orders" that looped the same phrases endlessly. The expected output was a detailed multi-paragraph legal reasoning section citing specific case facts, witnesses, and legal arguments.
  2. Judge also failed — the same model was used as the judge, and it couldn't parse its own JSON output format. Errors included "Unexpected token" and "No number after minus sign in JSON" — the model was emitting malformed JSON when asked to score outputs.
  3. Authentication issues — the first run hit "Missing Authentication header" because the local server doesn't require an API key but the judge was still trying to send one.

The core problem was that the 4-bit quantized Gemma-3-4B model, even after fine-tuning, wasn't producing coherent structured legal text in Nepali. The model was also being used as its own judge, which created a compounding failure.

Key takeaway: The 4-bit quantized Gemma-3-4B model failed to produce coherent legal reasoning in Nepali across all 5 evaluation runs. The model also failed as its own judge due to malformed JSON output. A larger model or improved quantization approach may be needed.
